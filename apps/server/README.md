# Kracken Messenger Server

## Проблемы, которые были исправлены:

1. **Несоответствие событий**: Фронтенд отправлял `'message'`, а сервер ожидал `'send_message'`
2. **Отсутствовало поле `room_id` в таблице messages** - сообщения не привязаны к комнатам
3. **Фронтенд не обрабатывал событие `'new_message'`** от сервера
4. **Отсутствовала аутентификация пользователей** при подключении
5. **Неправильная схема базы данных** для поддержки комнат

## Установка и запуск:

### 1. Установите зависимости:
```bash
npm install
```

### 2. Создайте файл .env:
```bash
cp env.example .env
```

Отредактируйте `.env` файл, добавив:
- `DATABASE_URL` - URL вашей PostgreSQL базы данных
- `JWT_SECRET` - секретный ключ для JWT токенов
- `EMAIL_USER` и `EMAIL_PASS` - для отправки email (опционально)

### 3. Инициализируйте базу данных:
```bash
node scripts/init-db.js
```

### 4. Создайте тестового пользователя:
```bash
node scripts/create-test-user.js
```

### 5. Запустите сервер:
```bash
node server.js
```

## Структура базы данных:

- `users` - пользователи
- `chat_rooms` - чат-комнаты
- `chat_room_participants` - участники комнат
- `messages` - сообщения (теперь с привязкой к комнатам)

## API endpoints:

- `POST /api/auth/login` - вход пользователя
- `POST /api/auth/register` - регистрация пользователя
- `POST /api/auth/verify-email` - подтверждение email

## Socket.IO события:

- `authenticate_with_token` - аутентификация по токену
- `join_room` - присоединение к комнате
- `send_message` - отправка сообщения
- `new_message` - новое сообщение (broadcast)
